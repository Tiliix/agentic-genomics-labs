Part 1

The active-learning loop — data, tools, steps & results

This lab is not a one-shot analysis: it is an experiment-design loop. The job is to find the genes whose knockout produces a strong phenotype while running as few experiments as possible — exactly the problem a real CRISPR screen faces.

① What data is used — and is it real?

The lab references real public Perturb-seq / CRISPR screens — Norman 2019, Adamson 2016, and the genome-wide Replogle 2022 — as the kind of readout you would wire in for production. To keep the active-learning loop runnable entirely offline, it ships with a pathway-clustered synthetic screen simulator that provides a known ground-truth gene→phenotype table.

A CRISPR knockout screen measures the phenotypic effect of disabling each gene one at a time; the simulator stands in for that assay by returning a noisy phenotype score per gene. Because the truth is known, the agent's choices can be scored objectively without any wet-lab work.

Why this matters scientifically. The simulator plants a known ground truth: genes are organized into pathway clusters, and two pathways (MAPK_proliferation and cell_cycle) are designated strong-hit pathways centered near an effect of 0.85, while the rest sit near 0.05. A gene counts as a "hit" when its true effect exceeds 0.5. This structure rewards an agent that, after finding one hit, hypothesizes that its pathway neighbors are also hits — the exact structure active learning should exploit.

Step 1 & 2 — the ground truth & the noisy assay screen_env.measure() 
def measure(self, gene: str) -> float:
    # Simulate one screen readout: true effect + Gaussian noise.
    base = self.truth.get(gene, 0.0)                 # hidden true phenotype effect
    reading = float(base + self.rng.normal(0.0, MEASUREMENT_NOISE_SD))
    self.measured.setdefault(gene, []).append(reading)   # log every measurement
    return reading

def is_hit(self, gene: str) -> bool:                 # scoring only (hidden from agent)
    return self.truth.get(gene, 0.0) >= HIT_THRESHOLD   # HIT_THRESHOLD = 0.5

for r in range(1, rounds + 1):
    batch = proposer.propose(state, batch_size, avoid_essential)   # 1. PROPOSE
    if not batch:

    for gene, score in scores.items():
        state.record(gene, score)                    # 3. UPDATE running hit list

    state.hypothesis = proposer.critique(state)      # 4. CRITIQUE steers next round
    cumulative.append(len(state.hits))

⑤ The output — and how to read it scientifically

Each round prints what was proposed, the cumulative hit count, and the revised hypothesis. The run ends with an agent-vs-random table:

OutputWhat it meansHow to use it
tested NHow many distinct genes have been screened so farThe "experiment budget" spent — fewer is better for the same yield
cumulative hitsTrue hits discovered up to this roundThe thing being maximized; compare the agent's curve to random's
new hits this roundGenes that crossed the hit threshold in this batchShows whether the latest hypothesis paid off
hypothesisThe critic's current read on which pathway looks strongestThe agent's plain-English reasoning for the next batch
hit advantageAgent hits minus random hits at the endThe headline number — the active-learning payoff

Reading it like a scientist:

  • Sample-efficiency is the metric: the agent wins not by finding more hits in the limit, but by finding them in fewer experiments — the same hits, sooner. On a real screen that is saved reagents, time and money.
  • Why the agent beats random: once a hit in MAPK_proliferation shows up, the critic steers the next batch toward untested members of that same pathway, harvesting a cluster of hits at once instead of sampling blindly.
  • Noise is realistic: because measure() adds Gaussian noise, a single readout can mislead — which is why repeated measurements and pathway-level evidence matter, just like biological replicates.
  • This mirrors BioDiscoveryAgent: the propose→measure→critique pattern is the published active-learning design for genetic-perturbation discovery — here reproduced at toy scale so the whole loop is auditable.
In-silico only. This lab performs experiment design in silico — it ranks which knockouts are worth running. Any real perturbation requires wet-lab validation and biosafety approval before a single guide is ordered. The prioritized genes are hypotheses, not findings.

Part 2

The agent — what it adds, how it’s built, how to read its reasoning

The loop above will run with a deterministic heuristic. The agent upgrades the propose and critique steps into an Azure OpenAI model that reasons about pathway structure — choosing intelligently which genes to knock out next, and explaining why.

Proposes intelligently

Picks the next batch to maximize hit discovery, balancing exploitation of promising pathways against exploration of new ones.

Avoids waste

Excludes genes already tested and, on request, known-essential genes whose dropout is uninformative for the phenotype.

Critiques & beats random

A critic step revises the hypothesis each round, and the whole run is benchmarked head-to-head against random selection.

① The “skills” the agent possesses

This agent's skills are the three moves of the active-learning loop. The proposer owns propose and critique; the environment owns measure. Swap the proposer (Azure OpenAI ↔ heuristic) and the same loop runs unchanged.

SkillWhat invoking it does
proposeChoose the next batch of untested gene knockouts, given the hypothesis, top observed effects, and candidate pool
measureScreen each proposed gene via env.measure_batch() — a noisy phenotype readout
critiqueRevise the working hypothesis about which pathways are most promising, steering the next propose
benchmarkBuilt-in: run the same loop with a random baseline and report the hit advantage

Part 3

From a toy screen to real-world functional genomics

This lab perturbs 62 genes against a known answer key. Real functional genomics is far larger and messier — and that is exactly where an agentic design loop earns its keep.

What “complex” looks like

  • Genome-wide CRISPR and Perturb-seq screens — ~20,000 genes, combinatorial double-knockouts, single-cell readouts
  • Noisy, expensive assays where each round of experiments takes weeks and real reagents
  • Perturbation-prediction models — GEARS, scGPT, Geneformer — that can stand in for or pre-screen the measure() step
  • Rich prior knowledge to retrieve: pathways, protein interactions, prior screens, literature
  • Hard constraints: known-essential genes, off-target risk, guide quality (CRISPOR), biosafety tiers

Where the agent pattern scales

  • Target validation — prioritize which genes to perturb first, cutting the experimental budget to reach the same hits
  • Model-in-the-loop — call GEARS/scGPT to predict perturbation outcomes, reserving wet-lab rounds for the most uncertain candidates
  • Active learning at scale — the same propose→measure→critique loop over 20,000 genes instead of 62
  • Literature grounding — connect proposals to published evidence, the way BioDiscoveryAgent does
  • TechBio platforms — Recursion, Arc Institute and similar run exactly this design-screen-learn cycle at industrial scale

Value by stakeholder

Functional genomicists

Fewer wasted knockouts per discovered hit; the agent proposes, ranks and documents batches so experts focus on screen design and edge cases.

Pharma / target validation

Faster, cheaper route from "which gene matters?" to a validated target — sample-efficiency translates directly into reagent and timeline savings.

Research teams

An auditable, hypothesis-generating loop that pairs in-silico ranking with mandatory wet-lab validation before anything is acted on.

The non-negotiable: guardrails

Scaling agentic perturbation design safely depends on the same principles this small lab already demonstrates:

  • In-silico design only — the agent ranks experiments; it never performs a real perturbation.
  • Mandatory wet-lab validation — every prioritized gene is a hypothesis until confirmed at the bench.
  • Biosafety review — real genome editing requires institutional biosafety approval and oversight before any guide is ordered.
  • Human-in-the-loop — the agent proposes and explains; a qualified scientist reviews and decides.
The big idea: the bottleneck in functional genomics is which experiments to run, not the sequencing. A scientist-supervised agent that proposes, measures, critiques and documents a screen — exactly what you saw in Lab 6 — is how that bottleneck starts to open, from a 62-gene demo to genome-wide, model-in-the-loop discovery.
