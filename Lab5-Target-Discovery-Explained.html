Part 1

The pipeline — data, tools, steps & ranked output

Before any AI, the lab is a standard target-discovery workflow: pull disease-associated genes from human genetics, layer on druggability and safety, then rank. Understanding it is the key to understanding what the agent automates.

① What data is used — and is it real?

The lab is built on the Open Targets Platform GraphQL API — a public resource (no API key) that integrates human genetics with target–disease evidence. The worked example is type 2 diabetes mellitus (EFO_0001360). The downloader queries the live API first, and a cached JSON snapshot is bundled so the lab still runs fully offline.

Open Targets is valuable because it aggregates many independent lines of evidence into one per-target score breakdown:

  • Genetic association — GWAS and curated disease-gene links (the human-genetics signal)
  • eQTL / colocalization — surfaced via RNA-expression & affected-pathway evidence (which tissue, which mechanism)
  • Known drugs — targets already drugged for this or related indications
  • Tractability — is the target druggable, and by what modality (antibody / small molecule / PROTAC)?
  • Safety — curated liabilities and adverse-event signals
Why this matters scientifically. A target backed by human genetic evidence — a variant that alters the gene and tracks with disease — is more likely to be causal than one nominated by expression or animal models alone. That is the de-risking premise of the whole pipeline. But the cached snapshot ships illustrative, representative scores, not authoritative ones: re-run online for current numbers.

⑤ The output — and how to read it scientifically

The pipeline produces one row per candidate target, plus a written memo. These are the signals that matter:

Reading it like a scientist:

  • Genetic support is the headline: targets whose disease link is backed by human genetics are more likely to succeed in the clinic — historically, genetically supported programs progress at roughly twice the rate of those without.
  • But genetics is not destiny: a strong genetic_association raises the prior that modulating the target affects disease — it does not guarantee efficacy or safety. Tractability, biology, dosing window and trial execution all still gate success.
  • Druggability gates the shortlist: a perfectly genetically-validated target with no tractable modality is a research problem, not a near-term drug program.
  • Safety flags re-order the ranking: curated liabilities can demote an otherwise attractive target.
Honesty note. This is a prioritization aid that sits alongside experimental validation — not a standalone go/no-go. The output triages hypotheses for literature and wet-lab follow-up; it does not decide which targets advance.

Part 2

The agent — what it adds, how it's built, how to read its memo

The queries above return raw, fragmented evidence. The agent wraps them in an Azure OpenAI model that assembles the evidence, scores each target across four axes, explains the rationale, and writes a transparent prioritization memo — while being forbidden from inventing evidence and required to flag the genetics≠efficacy caveat.

Assembles fragmented evidence

One tool call pulls association scores, tractability and safety into a single bundle the model can reason over.

Ranks transparently

Scores every target on four named axes, each cited to its evidence type — no opaque black-box ordering.

Explains the rationale

Writes a per-target rationale paragraph naming the specific scores that drove the ranking.

Flags the caveat

Required to state explicitly that genetic support is supporting evidence, not a guarantee of efficacy or safety.

① The "skills" the agent possesses

An agent's skills are the tools it's allowed to call. This agent has exactly one evidence tool — wrapping the Open Targets queries — plus the built-in ability to rank and write the memo, granted by its system prompt.

Code — the tool body (assemble an evidence bundle) src/rank_agent.py 
def get_target_evidence(efo_id, top=10):
    base = _load_cached(efo_id); source = "cache"   # prefer offline-safe snapshot
    if base is None:
        base = associated_targets(efo_id, size=max(top, 25)); source = "live"

    enriched = []
    for t in base["targets"][:top]:
        ens = t["ensemblId"]
        tract, safety = t.get("tractability", []), t.get("safety", [])
        if not tract or not safety:        # enrich live when possible, tolerate failure
            try:
                tract = tract or target_tractability(ens)
                safety = safety or target_safety(ens)
            except Exception: pass
        enriched.append({"symbol": t["symbol"], "datatypeScores": t["datatypeScores"],
                         "tractability": tract, "safetyLiabilities": safety, ...})
    return {"disease": base["disease"], "source": source, "targets": enriched}

The description is crucial — it's how the model "knows" what evidence the tool returns and how to reason over it. Good tool descriptions are the prompt engineering of agents.

② The code that defines the agent's behavior

Its scoring rationale — the system prompt SYSTEM_PROMPT 
You are a computational drug-discovery analyst. You prioritize drug targets for a
disease using ONLY the evidence returned by the get_target_evidence tool (Open Targets
Platform). You must:

1. Call get_target_evidence exactly once for the requested disease.
2. Score every returned target across four axes, each cited to its evidence type:
   - Human genetic support  -> datatypeScores.genetic_association
   - Tissue / mechanism specificity -> rna_expression and affected_pathway
     (eQTL/colocalization-derived); note tissue if present.
   - Tractability -> the tractability buckets (which modality is druggable?).
   - Safety -> safetyLiabilities (more/severe liabilities => lower score).
3. Produce a ranked, transparent target-prioritization memo in MARKDOWN with a
   summary, a ranked table, top-3 rationale paragraphs, and a "Caveats" section.
4. CRITICAL caveat you must state explicitly: human genetic association is SUPPORTING
   evidence and does NOT guarantee clinical efficacy. ... Recommend the memo be used to
   triage hypotheses for literature/wet-lab follow-up, not for go/no-go decisions.

Be concrete and quantitative. Never invent evidence not present in the tool output.
Its engine — the tool-calling loop run_agent() 
# First turn: the model decides to call the tool.
first = client.chat.completions.create(
    model=deployment, messages=messages,
    tools=TOOLS, tool_choice="auto", temperature=0.2)
msg = first.choices[0].message
messages.append(msg.model_dump(exclude_none=True))

# Execute any tool calls the model requested.
if msg.tool_calls:
    for call in msg.tool_calls:
        args = json.loads(call.function.arguments or "{}")
        result = get_target_evidence(efo_id=args.get("efo_id", efo_id),
                                     top=int(args.get("top", top)))
        messages.append({"role": "tool", "tool_call_id": call.id,
                         "name": call.function.name, "content": json.dumps(result)})

# Second turn: the model writes the memo from the tool output.
final = client.chat.completions.create(model=deployment, messages=messages, temperature=0.2)
return final.choices[0].message.content or ""

③ How to read the prioritization memo scientifically

The memo has a fixed, auditable shape. Learning to skim it is how you check what the agent did.

# Target Prioritization Memo — type 2 diabetes mellitus (EFO_0001360)
One-paragraph summary of the genetics-first shortlist ...

| Rank | Symbol | Overall | Genetic | Tractability | Safety flags |
|------|--------|---------|---------|--------------|--------------|
| 1    | PPARG  | 0.91    | 0.88    | small molecule | ...        |

## Top-3 rationale  — why each target ranks, citing genetic_association,
                     rna_expression/affected_pathway, tractability, safety.
## Caveats  — genetic support is SUPPORTING evidence, not a guarantee.
  • The ranked table — every target with its four-axis scores; the Genetic column is the de-risking signal.
  • The rationale paragraphs — for the top targets, the specific evidence types and scores that drove the rank.
  • The Caveats section — the mandatory, explicit statement that genetics ≠ efficacy or safety.
Scientific interpretation — the key habit: every score in the memo (e.g. a genetic_association of 0.88) must trace back to the get_target_evidence tool result, not to the model's imagination. The system prompt's rule "never invent evidence" plus tool-grounding is what makes the memo trustworthy — and the Caveats section is what keeps a high genetic score from being misread as a green light.

Part 3

From one disease to genetics-driven drug discovery at scale

This lab ranks targets for one disease from one evidence source. Real target discovery fuses many genetic modalities across whole portfolios — and that complexity is exactly where agentic AI earns its keep.

What "complex" looks like

  • Common-variant GWAS across hundreds of traits and biobank cohorts
  • Rare-variant / burden tests from exome and whole-genome sequencing (loss-of-function "human knockouts")
  • eQTL / pQTL colocalization to link a variant to the gene and protein it actually changes
  • Mendelian randomization to argue causality, not just correlation, and predict on-target effects
  • Reconciling tractability, safety, expression atlases, and competitive/IP landscape per target

Where the agent pattern scales

  • Evidence assembly — the same fetch→score→explain loop, over dozens of genetic sources instead of one
  • Transparent triage — rank thousands of target–disease pairs with cited, auditable rationales
  • Literature grounding — connect each candidate to published evidence with citations
  • Portfolio views — surface genetically-supported, druggable, low-liability targets across a pipeline
  • Natural-language access — ask "best genetically-supported, tractable targets for T2D" in English

The real-world landscape

Platforms & agentic AI

Open Targets (the open evidence backbone used here), PandaOmics / Insilico Medicine, BenevolentAI, Recursion, and Variant Bio build genetics- and AI-driven target-discovery engines that automate exactly this kind of evidence integration.

Genetics-driven pharma

The Regeneron Genetics Center and deCODE genetics (with Amgen) pioneered the strategy of nominating and de-risking targets from large-scale human-genetics cohorts before committing to programs.

Value by stakeholder

Pharma

De-risk the portfolio early: prioritize genetically-supported, tractable targets and kill weak hypotheses before expensive programs start.

Biotech

A small team can scan the genetic evidence landscape, defend a target thesis with cited scores, and move faster on a focused pipeline.

Researchers

Self-serve, plain-language target triage that turns fragmented public genetics into a ranked, explained hypothesis list.

The non-negotiable: guardrails

Scaling genetics-driven, agentic target discovery safely depends on the same principles this lab already demonstrates:

  • Evidence-grounded, never fabricated — every score traces to a tool result, with the source recorded.
  • Genetics ≠ efficacy or safety — genetic support raises the prior; it is not proof, and the memo must say so.
  • Validation, not autonomous go/no-go — the agent triages hypotheses for literature and wet-lab follow-up; humans decide what advances.
  • Human-in-the-loop — a qualified scientist reviews the ranking and the caveats before any commitment.
The big idea: the bottleneck in early drug discovery is picking the right target, and human genetics is the best-validated way to improve that bet. A scientist-supervised agent that assembles the genetic evidence, ranks transparently, explains its reasoning and flags the caveats — exactly what you saw in Lab 5 — is how that bet gets made faster, without mistaking a strong genetic score for a guarantee.
