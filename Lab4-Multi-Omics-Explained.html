Part 1

The integration pipeline — data, tools, steps & factors

Before any AI, the lab is a standard multi-omics integration workflow: several measurement "views" of the same samples, collapsed into a handful of shared latent factors. Understanding it is the key to understanding what the agent reasons over.

① What data is used — and what is "multi-omics"?

The lab is built around the MOFA+ CLL dataset — a classic multi-omics tutorial cohort of chronic lymphocytic leukaemia patients. The same samples are measured four different ways: mRNA expression DNA methylation drug response mutations. Each is a view — a distinct molecular layer on one shared set of patients. That is what "multi-omics" means: not one assay, but several complementary windows onto the same biology.

The downloader tries to fetch the real CLL bundle first. In this offline container the network is unavailable, so the script automatically falls back to a synthetic multi-omics matrix built from a shared latent structure — the same four view names, the same long-format shape.

Why this matters scientifically. The synthetic generator builds every view from a shared low-rank latent structure (a few hidden factors), so by construction the four omics layers co-vary through those factors — exactly the cross-view signal MOFA+ is designed to recover. The biology isn't real, but the covariation structure is genuine, making it a self-checking teaching dataset: drop in a real data/cll_long.csv and the identical run yields interpretable factors.

Code — the synthetic 4-view generator (shared latent structure) generate_synthetic() 
n_features = {"mRNA": 120, "Methylation": 100, "Drugs": 40, "Mutations": 30}
Z = rng.normal(size=(n_samples, n_factors))      # shared factors: samples x factors

for view, n_feat in n_features.items():
    W = rng.normal(size=(n_factors, n_feat))     # per-view loadings
    mask = rng.random(size=W.shape) < 0.4         # ~60% zeroed for sparsity
    X = (Z @ (W * mask)) + rng.normal(scale=0.5, size=(n_samples, n_feat))
    if view == "Mutations":  X = (X > 0.7).astype(float)   # binary calls
    elif view == "Methylation": X = np.tanh(X)            # M-value-like range

Output: data/cll_long.csv in long format with columns [sample, feature, view, value] — exactly the shape MOFA+ accepts.

⑤ The output — and how to read it scientifically

MOFA+ produces three linked artefacts. These are what matter:

ArtefactWhat it meansHow to use it
factors.csvLatent factors — shared axes of variation across all omics views (samples × factors)A factor groups correlated signals spanning DNA, RNA, methylation and drug response
weights/<view>.csvPer-view feature weights (features × factors); the signed loading of each feature on each factorDirection & size of a feature's contribution to a factor
top_features_per_factor.jsonThe top-|weight| features per factor, across all views, with signed weight and view of originThe agent's input — what drives each factor, and where the signal lives
ranked hypothesesCandidate-driver statements with evidence-strength scoresEach one traceable to a factor + feature weight; the actionable output
4
omics views integrated jointly
10
latent factors (default --n-factors)
15
top features kept per factor

Reading it like a scientist:

  • A factor is a shared axis of variation: it groups features that move together across DNA, RNA and methylation. A high-weight gene that appears across several views on the same factor is a candidate driver worth a closer look.
  • Sign and magnitude both matter: the signed weight tells you direction; a large absolute weight tells you how strongly that feature defines the factor.
  • Correlation across layers is NOT mechanism: a factor only says these signals co-vary — it does not say one causes another, or that the gene drives the disease. That gap is exactly what the agent is built to respect.
  • Every hypothesis is traceable: each candidate-driver statement points back to a specific factor and the feature weights it rests on, so a scientist can audit it against the source matrices.
Honesty note. The hypotheses this pipeline generates are exploratory. A MOFA+ factor surfaces covariation, not causation; any candidate driver requires independent statistical and experimental validation before it can be treated as biology.

Part 2

The agent — what it adds, how it’s built, how to read its reasoning

The pipeline above produces factors and weights — but not interpretation. The agent wraps those artefacts in an Azure OpenAI model that interprets each factor, connects its top features to biology, retrieves literature evidence, and ranks candidate-driver hypotheses — while being forbidden from claiming mechanism from correlation.

Interprets factors

Reads the top-weighted features of each latent factor and proposes what covariation pattern they might represent.

Connects to biology

Pulls supporting / contradicting literature from PubMed (or Azure AI Search) to ground each candidate driver.

Ranks & scores

Orders hypotheses and assigns an evidence-strength score from data support plus literature — never inventing either.

Refuses to claim mechanism

A hard rule: factors capture covariation, not causation — every item stays a tentatively-phrased hypothesis.

① The “skills” the agent possesses

This agent's skills are three reasoning steps applied to each factor: factor interpretation, evidence retrieval, and ranking. The first and third are granted by its system prompt; the middle one is a real tool call to a literature index.

SkillWhat invoking it does
factor interpretationRead the top features + signed weights of a factor and propose what it may represent
evidence retrievalQuery PubMed E-utilities (or Azure AI Search) for literature supporting or contradicting each candidate
rankingOrder hypotheses and assign an evidence-strength score in [0,1] from data + literature

Code — the correlation ≠ causation guardrail SYSTEM_PROMPT 
You are a cautious computational-biology research assistant.
You are given the top-weighted features of latent factors learned by
MOFA+ across several omics views (mRNA, methylation, drug response,
mutations) for a chronic lymphocytic leukaemia cohort.

HARD RULES — follow exactly:
1. MOFA+ factors capture COVARIATION, not causation. NEVER assert
   mechanism or causality from correlation. Every item is a HYPOTHESIS,
   phrased tentatively ("may", "could", "is consistent with").
2. Every hypothesis MUST cite its data basis: the factor name, the
   specific feature(s), their signed weight(s), and the view(s).
   Do not invent features or weights — use only those provided.
3. Assign an evidence_strength score in [0,1] from weight magnitude /
   how many views co-vary, AND whether literature supports it.
4. If literature is absent, say so; do not fabricate citations or PMIDs.

② How to read the agent’s ranked output

For each factor the agent returns strict JSON: a ranked list of hypotheses, each with caveats, data citations, an evidence score, and any supporting / contradicting literature. A human-readable hypotheses.md is written alongside it.

# Candidate-driver hypotheses (MOFA+ + Azure OpenAI)
## Factor1
### #1 (evidence 0.62)
Elevated expression of mRNA_017 may be consistent with the
methylation shift seen on this factor.
*Caveats:* correlational; could be confounded by cohort batch.
Data citations:
- `Factor1` / `mRNA_017` (mRNA) weight=1.84
- `Factor1` / `Methylation_044` (Methylation) weight=-1.21
  • rank + statement — the tentatively-phrased candidate driver, ordered most to least supported.
  • data_citations — the exact (factor, feature, view, weight) the claim rests on; this is the audit trail back into the MOFA+ output.
  • evidence_strength + evidence_rationale — a [0,1] score and a one-line explanation of how data support and literature combined to produce it.
Scientific interpretation — the key habit: read every hypothesis as a question, not an answer. Confirm each data_citation matches the weight in factors.csv / weights/<view>.csv, check the cited PMIDs are real, and treat a high score as "worth validating", never "proven". The hard rules in the system prompt — covariation not causation, cite the data, don't fabricate citations — are what make the output trustworthy enough for a scientist to act on.

Part 3

From a toy lab to real-world multimodal integration

This lab integrates four tidy views into ten factors. Real translational genomics fuses far more modalities — and that complexity is exactly where agentic AI earns its keep.

What true multimodal integration looks like

  • More modalities: DNA (WGS/WES), RNA, methylation, proteomics, single-cell, plus imaging (histopathology, radiology) and structured clinical data
  • Heterogeneous scales & likelihoods: continuous, count, binary and image-embedding views that must be modelled together
  • Missing data everywhere: not every assay is run on every patient — integration must tolerate gaps
  • Branching scientific decisions: batch correction, normalisation, factor count, which views to trust
  • Strict provenance: regulators and journals demand every parameter, version and input be reproducible

Where the agent pattern scales

  • Orchestration — the same interpret→retrieve→rank loop over dozens of views and thousands of features
  • Hypothesis triage — surface the handful of cross-modal signals worth a wet-lab follow-up
  • Evidence grounding — connect each candidate to the literature with real, checkable citations
  • Provenance & lineage — record which factor, weight and source backs every statement
  • Natural-language access — ask "what co-varies with drug resistance?" instead of writing integration code

The non-negotiable: guardrails

Scaling agentic multi-omics safely depends on the same principles this small lab already demonstrates:

  • Validation before belief — every hypothesis is exploratory until independent statistical and experimental work confirms it.
  • Correlation ≠ causation — factors capture covariation; the agent must never upgrade that to mechanism.
  • Full lineage — each claim cites its factor, feature, signed weight and view, plus real literature PMIDs.
  • Human review — the agent proposes and ranks; a qualified scientist reviews, decides, and owns the call.
The big idea: in multi-omics the bottleneck is interpretation — turning shared axes of variation into hypotheses a scientist can test — not the sequencing itself. A scientist-supervised agent that interprets factors, grounds them in evidence, and ranks them honestly — exactly what you saw in Lab 4 — is how that bottleneck starts to open, from a 4-view demo to true multimodal integration.
