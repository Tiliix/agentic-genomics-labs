Part 1

The classic pipeline — data, tools, steps & results

Before any AI, the lab is a standard, deterministic single-cell RNA-seq workflow. Understanding it is the key to understanding what the agent automates.

① What data is used — and is it real?

The lab is built around PBMC 3k, the classic Scanpy tutorial dataset: real single-cell RNA-seq of ~2,700 peripheral blood mononuclear cells from a healthy donor, sequenced on the 10x Genomics platform (Cell Ranger 1.1.0).

It is loaded directly via scanpy.datasets.pbmc3k(), which fetches the filtered gene-barcode matrix and returns a ready-to-use AnnData object. If the loader can't reach the internet, the script falls back to downloading the raw 10x .tar.gz and parsing the MTX triplet.

What the matrix actually is. Single-cell data is a genes × cells count matrix: each column is one cell, each row is one gene, each entry is how many transcripts of that gene were captured in that cell. It is overwhelmingly sparse — most genes are zero in most cells — because only a few thousand of the ~20,000 genes are detectably expressed per cell. This is real biology, not synthetic: the cells, the donor, and the counts are genuine, which is exactly why PBMC 3k is the reference dataset for learning the workflow.
Code — load PBMC3k via Scanpy, with a manual 10x fallback scripts/download_data.py 
def main():

② The pipeline at a glance

Four deterministic stages turn a raw sparse count matrix into labelled cell populations:

1
Quality control

Filter cells & genes; drop high-% mitochondrial (dying) cells and likely doublets.

→
2
Normalize + HVG + PCA

Library-size normalize, log1p, pick highly-variable genes, reduce with PCA.

→
3
Neighbors + Leiden

kNN graph in PCA space, UMAP, then Leiden community detection into clusters.

→
4
Marker genes

rank_genes_groups finds each cluster's top markers → cell-type annotation.

④ The steps, in code

Each stage is a small, readable Scanpy function. Toggle any snippet open.

Step 1 — Quality control filtering quality_control() 
# Basic count-based filtering of empty/low-complexity cells and rare genes.
sc.pp.filter_cells(adata, min_genes=MIN_GENES_PER_CELL)   # 200
sc.pp.filter_genes(adata, min_cells=MIN_CELLS_PER_GENE)   # 3

# Flag mitochondrial genes (human symbols start with "MT-").
adata.var["mt"] = adata.var_names.str.startswith("MT-")
sc.pp.calculate_qc_metrics(adata, qc_vars=["mt"], percent_top=None,
                           log1p=False, inplace=True)

# Remove probable doublets (very high gene counts) and dying cells (high mito).
adata = adata[adata.obs["n_genes_by_counts"] < MAX_GENES_PER_CELL, :].copy()  # 2500
adata = adata[adata.obs["pct_counts_mt"] < MAX_PCT_MITO, :].copy()           # 5.0%
Step 2 — Normalize, log1p, highly-variable genes, PCA normalize() + embed_and_cluster() 
adata.layers["counts"] = adata.X.copy()    # keep raw counts for later

# Normalize each cell to 10,000 total counts, then natural-log(1+x).
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)
adata.raw = adata                          # stash full gene set for DE

# Select highly variable genes, then scale + PCA.
sc.pp.highly_variable_genes(adata, n_top_genes=N_TOP_HVG, flavor="seurat")  # 2000
adata = adata[:, adata.var["highly_variable"]].copy()
sc.pp.scale(adata, max_value=10)
sc.tl.pca(adata, svd_solver="arpack", n_comps=N_PCS)   # 40 PCs
Step 3 — Neighbor graph + Leiden clustering embed_and_cluster() 
# Build the kNN graph in PCA space, then a UMAP embedding for visualization.
sc.pp.neighbors(adata, n_neighbors=N_NEIGHBORS, n_pcs=N_PCS)   # 15 / 40
sc.tl.umap(adata)

# Leiden community detection -> one integer label per cell.
sc.tl.leiden(
    adata,
    resolution=LEIDEN_RESOLUTION,   # 1.0 — higher = more, finer clusters
    flavor="igraph",
    n_iterations=2,
    directed=False,
)
Step 4 — Marker genes per cluster rank_markers() 
# Wilcoxon rank-sum test is the recommended default for scRNA-seq DE.
sc.tl.rank_genes_groups(adata, groupby="leiden", method="wilcoxon")

# Flatten Scanpy's structured result into a long DataFrame.
result = adata.uns["rank_genes_groups"]
groups = result["names"].dtype.names            # cluster labels
for cluster in groups:
    for rank in range(N_TOP_MARKERS):           # top 25 per cluster
        rows.append({"cluster": cluster, "rank": rank + 1,
                     "gene": result["names"][cluster][rank],
                     "log2fc": float(result["logfoldchanges"][cluster][rank]),
                     "pval_adj": float(result["pvals_adj"][cluster][rank])})

⑤ The output — and how to read it scientifically

The pipeline produces three artifacts: a UMAP coloured by cluster, a top-marker table per cluster, and the processed AnnData. The science lives in turning clusters into cell identities:

ArtifactWhat it containsHow to read it
umap_leiden.png2-D map of all cells, coloured by Leiden clusterSanity check — well-separated blobs suggest real, distinct populations
markers_top.csvTop-25 differentially expressed genes per clusterThe fingerprint of each cluster; canonical markers reveal identity
pbmc3k_processed.h5adFull AnnData: embeddings, clusters, DE resultsThe reproducible object every downstream step reads

From markers to cell types. Each cluster's top markers map to a known PBMC population:

Marker genesCell type
CD3D, IL7RT cells (CD4+ / CD8+)
MS4A1, CD79AB cells
CD14, LYZCD14+ Monocytes
FCGR3A, MS4A7FCGR3A+ Monocytes
NKG7, GNLYNK cells
FCER1A, CST3Dendritic cells

Reading it like a scientist:

  • Clusters are not cell types — yet. Leiden returns numbered groups. They only become biology once a marker fingerprint is matched to a canonical cell identity.
  • Resolution is a judgment call. LEIDEN_RESOLUTION directly controls how many clusters you get — too low merges distinct types, too high splits one type into artificial fragments. There is no single "correct" number.
  • Annotation is a judgment call too. Some clusters have clean, unambiguous markers; others sit between states (e.g. cycling vs. exhausted T cells) and reasonable experts can disagree.
  • Both choices ripple downstream. Every later result — proportions, differential abundance, trajectory — inherits the cluster boundaries and labels you set here.
Honesty note. The data is real, but the labels are inferred: a cluster called "NK cells" is a hypothesis supported by its markers, not a measured fact. The workflow is canonical; the annotations still require expert validation before they drive any biological conclusion.

Part 2

The agent — what it adds, how it's built, how to read its reasoning

The pipeline above produces numbered clusters and marker lists. The agent reads those markers and does the part that used to require a human poring over the literature: it names each cell type, scores its own confidence, justifies the call from specific genes, and then critiques its own labels — the GPTCelltype idea, plus a self-review pass.

Drives parameter choices

In the agent-driven variant, an LLM can propose QC thresholds and clustering resolution from the QC plots — instead of fixed defaults.

Annotates from markers

Maps each cluster's top marker genes to a canonical PBMC cell type, with a confidence score and a one-line justification.

Self-critiques

Re-reads its own labels and flags genuinely ambiguous clusters (cycling vs. exhausted, monocyte vs. dendritic) for human review.

① The "skills" the agent possesses

This agent's skills are two structured LLM behaviors over the marker table — annotate and critique — mirroring the GPTCelltype paper (Hou & Ji, 2024) and adding a review pass on top.

# Compact, deterministic prompt. We request strict JSON so it parses cleanly.
cluster_block = "\n".join(
    f"- Cluster {cid}: {', '.join(genes)}" for cid, genes in markers.items()
)

system_prompt = (
    "You are an expert single-cell genomics annotator. Given the top "
    "differentially expressed marker genes for each cluster, assign the most "
    "likely cell type. Be cautious: base labels strictly on canonical marker "
    "biology, and lower your confidence when markers are mixed or generic."
)
Code — it returns label + confidence + justification (strict JSON) annotate_clusters() 
user_prompt = (
    f"Tissue context: {TISSUE_CONTEXT}.\n\n"            # human PBMC
    f"Marker genes per cluster (ranked, most significant first):\n{cluster_block}\n\n"
    "For EACH cluster return a JSON object with this exact schema:\n"
    '{"annotations": [{"cluster": "<id>", "cell_type": "<label>", '
    '"confidence": <0.0-1.0>, "justification": "<one line citing the key '
    'markers>"}]}\n'
    "Use standard PBMC cell-type names (e.g. CD4+ T cells, CD8+ T cells, "
    "NK cells, B cells, CD14+ Monocytes, FCGR3A+ Monocytes, Dendritic cells, "
    "Megakaryocytes). Output JSON only."
)

resp = client.chat.completions.create(
    model=deployment,
    temperature=0.0,                          # deterministic labelling
    response_format={"type": "json_object"},
    messages=[{"role": "system", "content": system_prompt},
              {"role": "user", "content": user_prompt}],
)
annotations = json.loads(resp.choices[0].message.content)["annotations"]
Code — the self-critique pass flags ambiguity self_critique() 
# Guards against confident-but-wrong labels — the classic LLM failure mode.
user_prompt = (
    "Here are proposed annotations with their marker genes:\n"
    f"{json.dumps(review_input, indent=2)}\n\n"
    "For EACH cluster, decide whether the label is ambiguous. Pay special "
    "attention to commonly confused states, e.g.:\n"
    "  - cycling (MKI67, TOP2A) vs. exhausted (PDCD1, LAG3, HAVCR2) T cells\n"
    "  - CD14+ monocytes vs. dendritic cells (FCER1A, CST3)\n"
    "  - naive vs. memory T-cell subsets\n"

    '"alternative": "<other plausible cell type or null>", '
    '"reason": "<short explanation>"}]}\n'
)

② How to read the agent's reasoning output

Both passes are merged into one tidy table, then printed with flagged clusters surfaced for review. Learning to skim it is how you audit what the agent decided.

# results/cell_type_annotations.csv (one row per cluster)
cluster, cell_type,        confidence, justification,                      ambiguous, alternative
0,       CD4+ T cells,     0.93,       "IL7R, CD3D — canonical CD4 T",     False,     null
3,       NK cells,         0.88,       "NKG7, GNLY cytotoxic markers",     False,     null
6,       T cells (state?), 0.55,       "CD3D+ but MKI67 high",             True,      "cycling vs exhausted"

# console summary
  cluster  6: T cells (state?)     conf=0.55  <-- REVIEW (ambiguous)
  • cell_type + confidence — Pass 1's label and how sure the model is; low confidence is itself a signal.
  • justification — the specific markers the model used, so you can check its reasoning against canonical biology.
  • ambiguous + alternative — Pass 2's verdict; a REVIEW flag is the agent explicitly asking for a human.
Scientific interpretation — the key habit: treat every label as a justified hypothesis, not a fact. Read the justification markers, trust high-confidence clusters with clean canonical fingerprints, and prioritize the ambiguous-flagged clusters for manual inspection. The self-critique pass exists precisely because GPTCelltype-style annotation can be confidently wrong — the flag is what keeps a scientist in the loop before these labels drive any downstream biology.

Part 3

From a toy lab to real-world complexity

This lab clusters ~2,700 cells from one healthy donor. Real single-cell work spans millions of cells, many tissues, and added spatial context — and that scale is exactly where agentic AI earns its keep.

What "complex" looks like

  • Large atlases — millions of cells across donors, tissues and conditions (HCA-scale), with batch effects to integrate
  • Spatial transcriptomics — expression tied to physical location, so cell types must be read in tissue context
  • Many cell states, not just types — cycling, exhausted, activated, transitional — a continuum, not tidy boxes
  • Reference mapping, label transfer and harmonizing ontologies across studies
  • Annotation that must survive expert and clinical scrutiny

Where the agent pattern scales

  • Single-cell foundation models — scGPT, Geneformer and CellTypist bring learned priors over millions of cells
  • Automated, scalable annotation — the same annotate→critique loop, over hundreds of clusters instead of a handful
  • Ambiguity triage — surface only the clusters that genuinely need an expert, saving human time
  • Literature-grounded labels — justify each call from canonical markers and published evidence
  • Natural-language access — ask "which clusters are exhausted T cells?" instead of writing code

The reliability caveat

Benchmarks vary. Foundation models and LLM annotators are powerful but inconsistent: reported accuracy swings by tissue, cell-state granularity, and benchmark choice, and a model confident on PBMCs can falter on rare or transitional states. None is a drop-in replacement for expert judgment — they are accelerators that still require validation against canonical markers and orthogonal evidence.

Value by stakeholder

Researchers

First-pass annotation of large datasets in minutes, with justifications and an ambiguity shortlist — turning weeks of manual marker-checking into a review task.

Biotech

Consistent, scalable, auditable cell-typing across pipelines and studies — reproducible labels that hold up as datasets and headcount grow.

Clinicians

Interpretable, cited cell-composition summaries — provided humans stay in the loop and labels are validated, never used for diagnosis unaided.

The non-negotiable: guardrails

Scaling agentic annotation safely depends on the same principles this small lab already demonstrates:

  • Annotations need expert validation — every label is a hypothesis to confirm against canonical markers, not a measurement.
  • Self-critique by default — the agent must surface what it is unsure about, not hide it behind a confident label.
  • Human-in-the-loop — the agent proposes and flags; a qualified person reviews and decides identity.
  • Bounded autonomy — clinical or therapeutic use requires validation and explicit approval.
The big idea: in single-cell genomics the bottleneck is interpretation — turning clusters into trustworthy cell identities at scale. A scientist-supervised agent that annotates, justifies and critiques its own calls — exactly what you saw in Lab 3 — is how that bottleneck starts to open, from a 2,700-cell demo to million-cell atlases.
